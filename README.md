### Why merge conflicts occur

A merge conflict happens because the same file has been changed in two different branches and when it comes time to merge both files and the changes together, there will be a conflict.

## Publishing to GitHub

### Creating a remote repository

In this step, I'm setting up a public GitHub repository so that I can push my local repository into GitHub and verify my commit history and branches are visible online.

![Image](https://learn.nextwork.org/contented_magenta_shy_skunk/uploads/8bf3bcb5-a70d-418a-9c76-251365f69028_l5jsusg6)

### Pushing local commits to GitHub

The git push command uploads your local commits to a remote repository, making them available to others.

You need to set up a remote first because git push needs a destination—a specific URL or alias (like origin) telling Git where to send your changes. Without a remote, Git has no target to push to.
